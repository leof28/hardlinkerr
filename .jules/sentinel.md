## 2024-07-16 - Path Traversal Vulnerability in Deletion Endpoints
**Vulnerability:** Several API endpoints (like `/api/delete-file`, `/api/delete-series`, etc.) accepted absolute or relative paths directly from the request JSON and passed them directly to `os.remove` or `shutil.rmtree` without any validation. This allowed path traversal (e.g. using `../../`) or specifying arbitrary absolute paths (e.g. `/etc/passwd`) leading to arbitrary file deletion on the host.
**Learning:** The endpoints trusted user input representing file paths blindly. This pattern indicates a lack of centralized path validation before file system operations.
**Prevention:** Implement a robust `is_safe_path` utility that relies on `os.path.abspath` and `os.path.commonpath` to ensure that any target path strictly falls within one of the application's configured and allowed root directories. Always invoke this validation before passing any path to `os.remove` or `shutil.rmtree`.
## 2024-07-16 - Missing Webhook Authentication and Timing Attack Vulnerability
**Vulnerability:** The `/api/webhook/radarr` endpoint was completely missing authentication checks (did not verify `webhookSecret`), and the `/api/webhook/jellyfin` endpoint was vulnerable to timing attacks because it verified the secret using standard string comparison (`!=`).
**Learning:** Similar endpoints can easily drift out of sync leading to missing security checks. Furthermore, validating tokens or secrets via standard string equality operators exposes the comparison to timing attacks since it fails fast.
**Prevention:** Always verify that newly implemented or duplicated endpoints have consistent security checks applied. Always use constant-time comparison methods, such as `hmac.compare_digest` in Python, for verifying secrets, passwords, or authentication tokens.
## 2026-07-21 - API Secrets Exposure in Settings Endpoint
**Vulnerability:** The `/api/settings` endpoint was returning all configuration data, including highly sensitive API keys (`apiKey`, `sonarrApiKey`, `tmdbApiKey`, `webhookSecret`, `jellystatApiKey`), in plain text without any authentication. This allowed any unauthenticated user with local network access to retrieve these secrets.
**Learning:** Endpoints that handle configuration data must filter or mask sensitive fields before returning them to the client, especially in applications that lack robust authentication layers.
**Prevention:** Always mask sensitive fields (e.g., replacing with `********`) in `GET` responses and implement logic in the `POST` handler to restore the original values from the saved configuration if the masked string is received, preventing accidental overwrites while keeping the secrets hidden.
## 2025-05-20 - Symlink Path Traversal Vulnerability
**Vulnerability:** The `is_safe_path` function in `bridge.py` used `os.path.abspath` to validate if a target path was within an allowed directory. An attacker could bypass this check by creating a symlink inside an allowed directory that pointed to a sensitive directory outside of it. `os.path.abspath` normalizes paths but does not resolve symlinks, making the malicious path appear to be inside the allowed directory.
**Learning:** `os.path.abspath` is insufficient for preventing path traversal when symlinks are involved. `os.path.commonpath` checks against the unresolved path, leading to security bypasses.
**Prevention:** Always use `os.path.realpath` to fully resolve symbolic links to their absolute destinations before comparing paths to verify if they are within allowed boundaries.
